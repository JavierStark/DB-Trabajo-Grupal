-- Algoritmo de asignación (greedy)
FUNCTION F_PLAZAS(p_sede VARCHAR2)
  RETURN NUMBER AS
BEGIN
  RETURN v_capacidad - v_asignados;
END;
-- Trigger de migración de centro
CREATE TRIGGER TR_MIGRAR_CENTRO
AFTER UPDATE OF Sede_Codigo ON CENTRO
FOR EACH ROW BEGIN
  MIGRAR_CENTRO(:NEW.Codigo,
    :OLD.Sede_Codigo, :NEW.Sede_Codigo);
END;
-- Ventana de reubicación urgente
IF NOT (v_hora BETWEEN SYSDATE
    AND (SYSDATE + 1/24)) THEN
  RAISE_APPLICATION_ERROR(...);
END IF;
Arquitectura de seguridad
Seguridad multicapa
1
Roles de acceso — ROL_ESTUDIANTE · ROL_VOCAL · ROL_ACCESO
Cada rol accede únicamente a sus vistas y procedimientos. Permisos mínimos por defecto.
GRANT SELECT ON V_MIS_DATOS
  TO ROL_ESTUDIANTE;
2
VPD — Virtual Private Database
Política a nivel de fila: Usuario_BD = USER. Cada estudiante ve únicamente su propia fila en ESTUDIANTE.
DBMS_RLS.ADD_POLICY(
  object_name => 'ESTUDIANTE',
  policy_function => 'FN_VPD'
);
3
Usuarios dinámicos — PK_SEGURIDAD_PAU
Crea EST_<DNI> y VOC_<DNI> con contraseña aleatoria. Bloqueo tras 5 intentos · caducidad 30 días · validación DNI.
p_pass := DBMS_RANDOM.STRING('X',10);
GRANT ROL_ESTUDIANTE TO p_user;
